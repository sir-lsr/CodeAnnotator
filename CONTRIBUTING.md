# 贡献指南

感谢您对 CodeAnnotator 项目的关注！我们欢迎所有形式的贡献。

## 如何贡献

### 报告问题

如果您发现了 bug 或有功能建议，请通过 GitHub Issues 提交：

1. 检查问题是否已经存在
2. 创建新 Issue，提供清晰的问题描述
3. 如果是 bug，请提供复现步骤
4. 如果是功能建议，请说明使用场景

### 提交代码

1. Fork 本仓库
2. 创建功能分支 (`git checkout -b feature/AmazingFeature`)
3. 提交更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 创建 Pull Request

### 代码规范

#### 后端（Python）

- 遵循 PEP 8 代码规范
- 使用类型提示（Type Hints）
- 编写文档字符串（Docstrings）
- 保持函数简洁，单一职责

#### 前端（TypeScript/React）

- 使用 TypeScript 严格模式
- 遵循 React Hooks 最佳实践
- 组件命名使用 PascalCase
- 使用 ESLint 和 Prettier 格式化代码

### 提交信息规范

提交信息应清晰描述更改内容：

```
feat: 添加新功能
fix: 修复 bug
docs: 更新文档
style: 代码格式调整
refactor: 代码重构
test: 添加测试
chore: 构建过程或辅助工具的变动
```

### 开发环境设置

1. Fork 并克隆仓库
2. 按照 README.md 中的说明设置开发环境
3. 创建功能分支进行开发
4. 确保代码通过所有检查

### Pull Request 流程

1. 确保代码符合项目规范
2. 添加必要的测试
3. 更新相关文档
4. 确保所有测试通过
5. 提交 PR 时提供清晰的描述

### 行为准则

- 尊重所有贡献者
- 接受建设性批评
- 专注于对项目最有利的事情
- 对其他社区成员表示同理心

感谢您的贡献！
